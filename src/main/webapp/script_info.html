本遊戲使用 Scala 2.8.1 (程式語言) 之 Liftweb 2.4 (網頁架構) 製作。
遊戲中使用 Comet 固定連線機制，故不需要一段時間重連網頁，或類似 Ajax 背景重連網頁。
資料庫使用 MySQL。

什麼是 Scala 語言？
http://www.scala-lang.org/
Scala 語言是一種物件導向加函數式語言，執行於 Java 虛擬機(JVM)上。
Scala 語言擁有靜態語言嚴謹的型別檢查及動態語言的靈活性，
程式不管大小，用 Scala 語言寫起來都是很愉快的，
小的程式用 Scala 簡便的語言，不用太多行就可以解決，
而大的程式 Scala 也擁有完整的架構讓程式不致於太亂，
加上 Scala 語言本身執行速度快，逼近 Java 的效能，
惟 Scala 語言多樣式的寫法對於新手入門比較困難(加上需要會Java)。

什麼是 Liftweb 網頁架構？
http://liftweb.net/
Liftweb 又簡稱 Lift，算是 Scala 語言的網頁架構之一。
Liftweb 算是一種重量型的架構，它的功能豐富，但是耗費的資源也高。
Liftweb 網頁架構提供了多種的模式(寫法)，就算同樣用 Liftweb 開發的，
也可能因為寫法不同而迥然不同，然後 Liftweb 文件不完整，
而且各種寫法的混雜在一起，增加程式設計師撰寫的難度。

什麼是 Comet 架構？
一般網頁架構是每次按連結時，重新連線，傳送資料之後，斷線。
但是需要定期更新資料的，大多採用一段時間之後重新連線，
有些程式雖然沒有手動連線，但是系統仍然在背景一定時間重新讀取資料，
這種程式雖然好寫，但是間隔不容易調整，太短可能浪費頻寬，
太長的話，又要等到時間到了才更新，浪費時間。
Comet 則是一種固定連線式的架構，網頁一旦開啟，連線一直保留的，
如果伺服器有新資料了，可以馬上傳送給瀏覽器，不用等瀏覽器下一次重新連線。

如果你對 Scala、Liftweb 等等有興趣的人，可以進一步和我連絡。

為什麼登入登出會有問題？
在 Comet 架構之下，按登出並不會馬上登出，一般要等 10 至 60 秒。
如果這段時間登入，可能會登入到前一個的，因為前一個尚未登出完成。
另外就算登入成功，也有可能在 10 至 60 秒內被登出，必須要再重新登入。
